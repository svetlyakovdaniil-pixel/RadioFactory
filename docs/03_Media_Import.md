# Media Import

## Общий процесс

```text
Загрузка → Обработка → Проверка → Библиотека → Удаление исходника
```

## Интерфейс

Для музыки, видео и превью используются отдельные разделы. Поддерживаются Drag & Drop, массовая загрузка, фоновая обработка и кнопка `Отменить импорт`.

Пользователь всегда видит:

- сколько файлов загружено;
- сколько обработано;
- сколько осталось;
- текущий файл;
- ошибки и их причины.

Перезагрузка Dashboard не сбрасывает очередь и прогресс.

## Независимые очереди

```text
Media Processor
├── Music Processor
├── Video Processor
└── Thumbnail Processor
```

Они работают независимо и параллельно.

## Music Processor

- применяет fade-in и fade-out;
- длительность задаётся глобально в Workspace;
- начальное значение — 3 секунды;
- после успеха сохраняет обработанный файл;
- исходник удаляет.

## Video Processor

- конвертирует видео под утверждённые параметры LIVE;
- проверяет, что файл читается и не повреждён;
- проверяет успешность конвертации;
- сохраняет обработанный файл;
- удаляет исходник.

Не проверяются аудио, соотношение сторон, зацикливаемость и длительность.

## Thumbnail Processor

Проверяет только разрешение `1280×720`. Другой файл отклоняется с понятной ошибкой.

## Отмена

Текущий файл завершается, новые не запускаются, уже добавленные остаются, необработанные временные файлы удаляются.

## Дубликаты

Одинаковое имя в одной библиотеке запрещено. Новый файл не заменяет старый и не переименовывается автоматически.

## Готовая медиатека

Доступны список, поиск, сортировка, одиночное и массовое удаление, предпросмотр и переименование. Новое имя должно быть уникальным.

## Приоритет

Активные LIVE всегда имеют более высокий приоритет. Диспетчер ресурсов снижает скорость или параллелизм фоновой обработки, если она может повлиять на эфиры.
