# LIVE Lifecycle

## Запуск

Пользователь видит пошаговый процесс:

1. Проверка Station.
2. Проверка музыки.
3. Проверка loop-видео.
4. Проверка превью.
5. Получение названия.
6. Получение описания.
7. Создание LIVE на YouTube.
8. Назначение превью.
9. Назначение плейлиста.
10. Запуск FFmpeg.
11. Проверка сигнала YouTube.
12. Статус LIVE.

Каждый этап показывает ожидание, успех или конкретную ошибку.

## Завершение

1. Завершить FFmpeg.
2. Завершить LIVE на YouTube.
3. Добавить запись в плейлист.
4. Сохранить ошибки и историю перезапусков.
5. Очистить временные данные этого LIVE.
6. Проверить очистку.
7. Запустить следующий LIVE.

## Изолированная очистка

У каждого LIVE своя временная рабочая область. Очистка одной Station не затрагивает другие Station, очереди импорта и постоянные библиотеки.

## Ошибка очистки

Система делает 3 попытки. Если не получилось:

- останавливается только эта Station;
- остальные продолжают работать;
- Station получает статус `Ошибка`;
- отправляется Telegram-уведомление;
- система ждёт вмешательства пользователя.

Безобидный остаточный файл не блокирует запуск. Конфликтующий lock-файл, живой FFmpeg или активное состояние старого broadcast блокируют следующий LIVE.
