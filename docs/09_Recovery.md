# Recovery

## Политика

Сначала автоматическое безопасное восстановление, затем вмешательство пользователя.

## Краткий разрыв

- менее 3 минут — продолжить тот же LIVE;
- 3 минуты и более — завершить старый LIVE, Cleanup, создать новый.

## Музыка

При восстановлении того же LIVE текущий трек продолжается с максимально точной позиции прерывания.

## Неудача одной Station

После повторных попыток Station переходит в Error, отправляется уведомление, остальные Station продолжают работу.

## Перезагрузка сервера

Восстанавливаются только ранее работавшие Station. Ручные остановки сохраняются. Запуск идёт через очередь с учётом ресурсов.

## Ошибка в очереди

После нескольких попыток Station переводится в Error, но очередь остальных продолжается. Решение №10 закрепляет три попытки в общем механизме восстановления; единообразие счётчиков требует отдельного уточнения.

## Диск

Останавливаются только затронутые Station. После освобождения места выполняется автоматическое восстановление. При неудаче — Error.

## Глобальная авария

Общая причина оформляется на уровне Workspace. Точные условия общей остановки ещё не формализованы.
